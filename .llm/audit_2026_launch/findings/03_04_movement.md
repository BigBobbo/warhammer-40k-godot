## Top 3 launch-blocker Movement gaps

1. **AIRCRAFT minimum-move 20" + board-edge end restriction not enforced (M14, M15).** Wazbom Blastajet (active Orks roster) has the AIRCRAFT keyword and `move=20` but the engine accepts any move 0–20" and any final position. Adding more aircraft (any faction) compounds. Cite: `MovementPhase.gd:137-170` (only pivot uses `is_aircraft`); no other AIRCRAFT branch in MovementPhase.
2. **TOWERING / Ruin upper-floor occupancy not enforced (M13, M17).** Movement phase has no rule preventing a non-FLY VEHICLE/MONSTER from ending on a Ruin floor, or for restricting upper-floor placement to FLY/INFANTRY (per 10e). LoS calculator knows about TOWERING; the movement controller does not. Launch-blocking when adding any Knight / Hierodule / large vehicle to a roster that includes Ruins (every standard layout).
3. **BEAST / SWARM cannot traverse Ruins (M12).** Hard-coded keyword list in `TerrainManager._add_sample_walls_to_terrain` only mentions INFANTRY/VEHICLE/MONSTER; BEAST and SWARM default to "blocked" by `can_unit_cross_wall`. The launch tier-3 factions (Tyranids gaunts, GSC rockgrubs, Aeldari Warp Spiders sans flying stem) are blocked from playing on Ruin maps.

## Top 3 invisible features

1. **Hover datasheet ability has zero engine handler (M16).** Wazbom Blastajet's "Hover" alternative move stat exists in the Wahapedia datasheet but is unreachable — there is no UI affordance, no datasheet-ability handler in `UnitAbilityManager`, no toggle in `MovementController`. Function exists at the data level (datasheet text); no code at all in the engine.
2. **Da Jump placement skips coherency / board-bounds / ER / center-vs-edge correctness (M23).** The 2026-05 audit closed T-105 against a pin test only. Live placement allows: model exactly 9.0" from an enemy (strict `<` instead of `≤`), models off the board, models on impassable terrain, models in broken coherency, models in ER of an enemy. Players can use Da Jump to teleport into illegal positions and the engine accepts them.
3. **Embark validation lacks 3"-range and ER checks at the action layer (M25).** UI prompt path is safe, but EMBARK_UNIT actions submitted outside the prompt (network desync replay, MCP scripting, future AI player) embark from any distance, including inside an enemy's engagement range. Pure validation gap.
