Controller Controls Map

Every point where a player gives input on a game controller, and exactly what each button does at that instant — across menus, pre-battle, and all seven battle phases of the Warhammer 40,000 Battle Simulator.

The button tables are generated straight from PadRouter's live hint-set constants, so a control listed here is a control the code actually renders. A drift-check test fails CI if the code changes without this document being updated — so these controls only change when we intend them to.

How to read this

The controls are contextual: the same button means different things in different states.

  • The pad only drives the game while it is the active device. Any joypad press claims control (the on-screen hint bar appears); any mouse move or key press hands control back to keyboard & mouse.
  • LB / RB are the only unit-switcher. They cycle the acting unit in every phase. Unit lists on the right are deliberately NOT walkable with the D-pad or stick — cycling is the bumpers' job alone.
  • The D-pad is context-dependent. With nothing selected it enters panel focus; with a phase sub-menu open (targets, weapons, move mode, deploy rows) it drives that instead.
  • Each state below lists its buttons, whether the right-hand panel can be driven on the pad, and any gaps. The "Right panel on pad?" badge is the answer to "can I control the menu on the right here?"

✚Move between tabs (Audio / Visual / Gameplay / Controls) and controls

Right panel on pad: The menu is modal and traps pad focus until closed

  • Full controller navigation; Close is default-focused.
  • The Controls tab shows a read-only controller reference.

Controller coverage gaps

Where a controller can't (yet) do something. Pad blocker = mouse/keyboard can, pad can't. No human path = AI-only for every input device.

Pad blocker

Shooting HUD buttons unreachable while a shooter is armed

Grenade, Perform/Start Secondary Action, Burn Objective, Clear All, Undo Last — the D-pad is captured by target/weapon stepping so panel-focus entry never fires. Only B (deselect) escapes, losing the shooting context.

ShootingController.gd:3948-3966 · PadRouter.gd:419-444 — code-traced, not yet runtime-confirmed

Pad gap

Stratagem panel has no dedicated pad affordance

Opened only by the S key or the bottom-bar button; no hint-bar chip. Technically reachable by focusing the button, but undiscoverable and pre-empted by phase steppers in most phases.

Main.gd:5847-5849 · StratagemPanel.gd

Pad gap

Deployment: can't reposition an already-placed model

The mouse Shift+click pick-up/drop has no pad equivalent during placement — X and B only undo. A pad player must undo and re-place.

DeploymentController.gd:119-146

Pad gap

Datasheet: pad-B doesn't close it, and it can't scroll

Only Y toggles it closed on pad (Esc on keyboard). A datasheet taller than the fixed card is clipped with no scroll (pad or mouse).

DatasheetModal.gd:18-19 · Main.gd:5829-5832

Pad gap

Can't bind actions to gamepad buttons

The keybinding remap UI only captures keyboard keys; the pad scheme is fixed and non-rebindable.

SettingsMenu.gd:831
